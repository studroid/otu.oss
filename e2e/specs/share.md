# 공유 테스트 계획

## 개요

OTU 앱의 페이지 공유 기능 테스트 계획입니다. 공개/비공개 전환과 공유 URL 접근을 포함합니다.

## 전제 조건

- 테스트 사용자로 로그인된 상태
- 공유 테스트용 페이지 존재

## 테스트 시나리오

### 1. 공유 상태 전환

**Seed:** `e2e/seed.spec.ts`

#### 1.1 페이지 공개로 설정

**Steps:**

1. 페이지 에디터에서 공유 버튼/메뉴 클릭
2. "공개" 옵션 선택
3. 변경 사항 저장

**Expected Results:**

- 페이지 상태가 "공개"로 변경됨
- 공유 URL이 생성되거나 표시됨
- 공유 아이콘/상태 표시 변경

#### 1.2 페이지 비공개로 설정

**Steps:**

1. 공개 상태의 페이지에서 공유 버튼/메뉴 클릭
2. "비공개" 옵션 선택
3. 변경 사항 저장

**Expected Results:**

- 페이지 상태가 "비공개"로 변경됨
- 기존 공유 URL 비활성화
- 공유 아이콘/상태 표시 변경

### 2. 공유 URL 접근

**Seed:** `e2e/seed.spec.ts`

#### 2.1 공개 페이지 URL 접근 (비로그인)

**Steps:**

1. 로그아웃 상태에서 공개 페이지의 공유 URL로 이동
2. 페이지 로드 대기

**Expected Results:**

- 페이지 내용이 읽기 전용으로 표시됨
- 로그인 없이 접근 가능
- 편집 기능 비활성화

#### 2.2 비공개 페이지 URL 접근 (비로그인)

**Steps:**

1. 로그아웃 상태에서 비공개 페이지 URL로 이동
2. 페이지 로드 대기

**Expected Results:**

- 접근 거부 또는 "페이지를 찾을 수 없음" 표시
- 로그인 페이지로 리디렉션 또는 오류 메시지

#### 2.3 공유 URL 복사

**Steps:**

1. 공개 페이지에서 공유 메뉴 열기
2. "URL 복사" 버튼 클릭

**Expected Results:**

- 클립보드에 공유 URL 복사됨
- 복사 완료 피드백 표시 (토스트 메시지 등)

### 3. 공유 페이지 표시

**Seed:** `e2e/seed.spec.ts`

#### 3.1 공유 페이지 목록 확인

**Steps:**

1. 페이지 목록에서 공유 상태 확인

**Expected Results:**

- 공개 페이지는 공유 아이콘/표시가 있음
- 비공개 페이지는 표시 없음

## 테스트 환경

- 로그인 필요: 공유 설정 시 예, 공개 페이지 접근 시 아니오
- 공유 기능 활성화 필요
