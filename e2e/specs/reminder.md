# 알람(리마인더) 테스트 계획

## 개요

OTU 앱의 페이지 알람 기능 테스트 계획입니다. 알람 설정, 활성화/비활성화를 포함합니다.

## 전제 조건

- 테스트 사용자로 로그인된 상태
- 알람 테스트용 페이지 존재
- 알람 권한 허용 상태

## 테스트 시나리오

### 1. 알람 설정

**Seed:** `e2e/seed.spec.ts`

#### 1.1 페이지에 알람 추가

**Steps:**

1. 페이지 에디터에서 알람 버튼/메뉴 클릭
2. 알람 날짜 및 시간 선택
3. 알람 저장

**Expected Results:**

- 알람이 페이지에 설정됨
- 알람 아이콘/표시가 페이지에 나타남
- 알람 목록에 추가됨

#### 1.2 알람 수정

**Steps:**

1. 알람이 설정된 페이지에서 알람 메뉴 열기
2. 알람 날짜/시간 수정
3. 변경 사항 저장

**Expected Results:**

- 알람 시간이 업데이트됨
- 수정된 시간이 표시됨

#### 1.3 알람 삭제

**Steps:**

1. 알람이 설정된 페이지에서 알람 메뉴 열기
2. 알람 삭제/비활성화 옵션 선택
3. 삭제 확인

**Expected Results:**

- 알람이 제거됨
- 알람 아이콘/표시 사라짐
- 알람 목록에서 제거됨

### 2. 알람 활성화/비활성화

**Seed:** `e2e/seed.spec.ts`

#### 2.1 알람 비활성화

**Steps:**

1. 활성화된 알람이 있는 페이지 열기
2. 알람 토글 또는 비활성화 버튼 클릭

**Expected Results:**

- 알람이 비활성화 상태로 변경됨
- 비활성화 표시 (회색 아이콘 등)
- 알람 시간에 알림 발송되지 않음

#### 2.2 알람 재활성화

**Steps:**

1. 비활성화된 알람이 있는 페이지 열기
2. 알람 토글 또는 활성화 버튼 클릭

**Expected Results:**

- 알람이 활성화 상태로 변경됨
- 활성화 표시 (컬러 아이콘 등)

### 3. 알람 목록 관리

**Seed:** `e2e/seed.spec.ts`

#### 3.1 알람 목록 보기

**Steps:**

1. 알람 관리 페이지/섹션으로 이동
2. 설정된 알람 목록 확인

**Expected Results:**

- 모든 설정된 알람이 목록에 표시됨
- 각 알람에 연결된 페이지 정보 표시
- 알람 시간 및 상태 표시

#### 3.2 알람 목록에서 페이지 이동

**Steps:**

1. 알람 목록에서 특정 알람 클릭
2. 연결된 페이지로 이동

**Expected Results:**

- 해당 페이지 에디터로 이동
- 알람 정보 확인 가능

## 테스트 환경

- 로그인 필요: 예
- 알람 권한 필요: 예 (브라우저 알림 권한)
- 시간 관련 테스트 시 주의 필요
