# 페이지 CRUD 테스트 계획

## 개요

OTU 앱의 메모 페이지 생성, 읽기, 수정, 삭제 기능 테스트 계획입니다.

## 전제 조건

- 테스트 사용자로 로그인된 상태
- 기존 페이지가 없거나 테스트용 페이지가 준비된 상태

## 테스트 시나리오

### 1. 페이지 생성

**Seed:** `e2e/seed.spec.ts`

#### 1.1 새 페이지 생성

**Steps:**

1. `/home` 페이지에서 새 페이지 생성 버튼 클릭
2. 에디터가 열리기를 대기

**Expected Results:**

- 새 페이지가 생성됨
- URL이 `/home/page/[새로운ID]` 형태로 변경됨
- 빈 에디터가 표시됨
- 페이지 목록에 새 페이지 추가됨

#### 1.2 제목 입력

**Steps:**

1. 새 페이지 생성
2. 제목 입력 필드에 "테스트 페이지 제목" 입력
3. 본문 영역 클릭 (포커스 이동)

**Expected Results:**

- 제목이 저장됨
- 페이지 목록에서 제목이 표시됨

### 2. 페이지 편집

**Seed:** `e2e/seed.spec.ts`

#### 2.1 텍스트 입력

**Steps:**

1. 기존 페이지 열기
2. 에디터 본문에 "테스트 내용입니다" 입력

**Expected Results:**

- 입력한 텍스트가 에디터에 표시됨
- 자동 저장 동작 (저장 표시 나타남)

#### 2.2 자동 저장 확인

**Steps:**

1. 페이지에 내용 입력
2. 2-3초 대기 (자동 저장 간격)
3. 페이지 새로고침

**Expected Results:**

- 새로고침 후에도 입력한 내용이 유지됨

#### 2.3 블록 에디터 기능

**Steps:**

1. 에디터에서 `/` 입력하여 슬래시 메뉴 열기
2. 헤딩, 리스트 등 블록 타입 선택

**Expected Results:**

- 슬래시 메뉴가 표시됨
- 선택한 블록 타입이 적용됨

### 3. 페이지 삭제

**Seed:** `e2e/seed.spec.ts`

#### 3.1 페이지 삭제

**Steps:**

1. 페이지 목록에서 삭제할 페이지 선택
2. 페이지 메뉴 열기 (케밥 메뉴 또는 우클릭)
3. 삭제 옵션 선택
4. 확인 다이얼로그에서 삭제 확인

**Expected Results:**

- 페이지가 목록에서 제거됨
- 휴지통으로 이동하거나 영구 삭제됨

### 4. 페이지 목록 표시

**Seed:** `e2e/seed.spec.ts`

#### 4.1 페이지 목록 로드

**Steps:**

1. `/home` 페이지로 이동
2. 페이지 목록 영역 확인

**Expected Results:**

- 사용자의 페이지 목록이 표시됨
- 각 페이지의 제목이 보임
- 페이지 클릭 시 해당 페이지로 이동

## 테스트 환경
