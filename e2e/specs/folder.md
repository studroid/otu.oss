# 폴더 시스템 테스트 계획

## 개요

OTU 앱의 폴더 생성, 관리, 페이지 할당 기능 테스트 계획입니다.

## 전제 조건

- 테스트 사용자로 로그인된 상태
- 테스트용 페이지가 최소 1개 존재

## 테스트 시나리오

### 1. 폴더 CRUD

**Seed:** `e2e/seed.spec.ts`

#### 1.1 새 폴더 생성

**Steps:**

1. `/home` 페이지의 사이드바에서 폴더 영역 확인
2. 새 폴더 생성 버튼 클릭
3. 폴더 이름 "테스트 폴더" 입력
4. 확인/저장

**Expected Results:**

- 새 폴더가 사이드바 폴더 목록에 추가됨
- 폴더 이름이 "테스트 폴더"로 표시됨

#### 1.2 폴더 이름 변경

**Steps:**

1. 기존 폴더 선택
2. 폴더 메뉴 열기 (우클릭 또는 케밥 메뉴)
3. 이름 변경 옵션 선택
4. 새 이름 "수정된 폴더" 입력
5. 확인

**Expected Results:**

- 폴더 이름이 "수정된 폴더"로 변경됨

#### 1.3 폴더 삭제

**Steps:**

1. 삭제할 폴더 선택
2. 폴더 메뉴 열기
3. 삭제 옵션 선택
4. 확인 다이얼로그에서 삭제 확인

**Expected Results:**

- 폴더가 목록에서 제거됨
- 폴더 내 페이지는 "미분류"로 이동 또는 유지

### 2. 페이지-폴더 할당

**Seed:** `e2e/seed.spec.ts`

#### 2.1 페이지를 폴더로 이동

**Steps:**

1. 페이지 목록에서 페이지 선택
2. 드래그 앤 드롭으로 폴더로 이동
   또는
3. 페이지 메뉴에서 "폴더 이동" 선택
4. 대상 폴더 선택

**Expected Results:**

- 페이지가 선택한 폴더 내로 이동됨
- 폴더 확장 시 해당 페이지가 표시됨

#### 2.2 폴더에서 페이지 제거

**Steps:**

1. 폴더 내 페이지 선택
2. 페이지를 폴더 밖으로 드래그
   또는
3. 페이지 메뉴에서 "폴더에서 제거" 선택

**Expected Results:**

- 페이지가 폴더에서 제거됨
- 페이지는 미분류 영역으로 이동

### 3. 폴더 네비게이션

**Seed:** `e2e/seed.spec.ts`

#### 3.1 폴더 확장/축소

**Steps:**

1. 폴더 클릭하여 확장
2. 폴더 내 페이지 목록 확인
3. 다시 폴더 클릭하여 축소

**Expected Results:**

- 확장 시 폴더 내 페이지 목록 표시
- 축소 시 페이지 목록 숨김
- 확장/축소 상태가 유지됨

#### 3.2 폴더 URL 접근

**Steps:**

1. 폴더가 있는 상태에서 해당 폴더 URL로 직접 접근

**Expected Results:**

- 해당 폴더가 선택된 상태로 표시됨
- 폴더 내 페이지 목록이 표시됨
