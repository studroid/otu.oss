# 검색 테스트 계획

## 개요

OTU 앱의 검색 기능 테스트 계획입니다. 키워드 기반 검색과 URL 상태 관리를 포함합니다.

## 전제 조건

- 테스트 사용자로 로그인된 상태
- 검색 가능한 페이지가 여러 개 존재

## 테스트 시나리오

### 1. 기본 검색

**Seed:** `e2e/seed.spec.ts`

#### 1.1 키워드 검색

**Steps:**

1. `/home` 페이지에서 검색 입력 필드 찾기
2. 검색어 "테스트" 입력
3. Enter 키 누르기 또는 검색 버튼 클릭

**Expected Results:**

- 검색 결과 목록이 표시됨
- "테스트" 키워드가 포함된 페이지만 필터링됨
- URL에 검색 쿼리 파라미터 반영

#### 1.2 검색 결과 없음

**Steps:**

1. 검색 입력 필드에 존재하지 않는 키워드 입력 (예: "zxcvbnm12345")
2. 검색 실행

**Expected Results:**

- "검색 결과 없음" 메시지 표시
- 빈 결과 목록

#### 1.3 검색어 초기화

**Steps:**

1. 검색어를 입력하여 검색 결과 표시 상태
2. 검색어 초기화 버튼 클릭 (X 버튼)

**Expected Results:**

- 검색어가 비워짐
- 전체 페이지 목록으로 복원
- URL에서 검색 쿼리 제거

### 2. URL 상태 관리

**Seed:** `e2e/seed.spec.ts`

#### 2.1 URL로 검색 상태 복원

**Steps:**

1. `/home?search=테스트` URL로 직접 이동
2. 페이지 로드 완료 대기

**Expected Results:**

- 검색 입력 필드에 "테스트" 표시
- 검색 결과가 필터링된 상태로 표시

#### 2.2 검색 상태 브라우저 뒤로가기

**Steps:**

1. 전체 페이지 목록 보기
2. 검색어 입력하여 검색 수행
3. 브라우저 뒤로가기

**Expected Results:**

- 검색 전 상태로 복원
- 검색어 필드 비워짐

### 3. 검색 UX

**Seed:** `e2e/seed.spec.ts`

#### 3.1 실시간 검색 (디바운스)

**Steps:**

1. 검색 입력 필드에 빠르게 "테스트" 타이핑
2. 타이핑 완료 후 잠시 대기

**Expected Results:**

- 타이핑 중에는 검색이 실행되지 않음
- 타이핑 완료 후 디바운스 시간 후 검색 실행
- 불필요한 API 호출 없음

#### 3.2 검색 결과 클릭

**Steps:**

1. 검색 실행하여 결과 표시
2. 검색 결과 중 하나의 페이지 클릭

**Expected Results:**

- 해당 페이지 에디터로 이동
- 검색 상태 유지 또는 초기화 (UX 정책에 따라)
