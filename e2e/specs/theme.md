# 테마 테스트 계획

## 개요

OTU 앱의 테마 전환 기능 테스트 계획입니다. 회색/흰색/검정 테마를 지원합니다.

## 전제 조건

- 테스트 사용자로 로그인된 상태 (또는 비로그인 상태에서도 테스트 가능)

## 테스트 시나리오

### 1. 테마 전환

**Seed:** `e2e/seed.spec.ts`

#### 1.1 밝은 테마 (흰색) 적용

**Steps:**

1. 설정 메뉴/페이지로 이동
2. 테마 옵션에서 "밝은" 또는 "흰색" 선택

**Expected Results:**

- 전체 UI가 밝은 테마로 변경됨
- 배경색이 흰색/밝은 색으로 변경
- 텍스트가 어두운 색으로 표시

#### 1.2 어두운 테마 (검정) 적용

**Steps:**

1. 설정 메뉴/페이지로 이동
2. 테마 옵션에서 "어두운" 또는 "검정" 선택

**Expected Results:**

- 전체 UI가 어두운 테마로 변경됨
- 배경색이 검정/어두운 색으로 변경
- 텍스트가 밝은 색으로 표시

#### 1.3 회색 테마 적용

**Steps:**

1. 설정 메뉴/페이지로 이동
2. 테마 옵션에서 "회색" 선택

**Expected Results:**

- 전체 UI가 회색 테마로 변경됨
- 배경색이 회색 계열로 변경

### 2. 테마 지속성

**Seed:** `e2e/seed.spec.ts`

#### 2.1 새로고침 후 테마 유지

**Steps:**

1. 특정 테마 선택 (예: 어두운 테마)
2. 페이지 새로고침
3. 현재 테마 확인

**Expected Results:**

- 새로고침 후에도 선택한 테마 유지
- 로컬 스토리지 또는 서버에 테마 설정 저장됨

#### 2.2 다른 페이지에서 테마 유지

**Steps:**

1. 특정 테마 선택
2. 다른 페이지로 이동 (예: 홈 → 에디터)
3. 테마 확인

**Expected Results:**

- 모든 페이지에서 동일한 테마 적용
- 페이지 전환 시 테마 깜빡임 없음

### 3. 시스템 테마 연동

**Seed:** `e2e/seed.spec.ts`

#### 3.1 시스템 테마 자동 적용 (옵션)

**Steps:**

1. 설정에서 "시스템 테마 따르기" 옵션 선택 (있는 경우)
2. OS 다크 모드 설정 변경

**Expected Results:**

- 시스템 테마에 맞춰 앱 테마 자동 변경
- 시스템 밝은 모드 → 앱 밝은 테마
- 시스템 어두운 모드 → 앱 어두운 테마

### 4. 테마 UI 요소

**Seed:** `e2e/seed.spec.ts`

#### 4.1 에디터 테마 적용

**Steps:**

1. 특정 테마 선택
2. 에디터 페이지로 이동
3. 에디터 영역 테마 확인

**Expected Results:**

- 에디터 배경색이 테마에 맞게 변경
- 에디터 텍스트 색상이 테마에 맞게 변경
- 슬래시 메뉴, 툴바 등도 테마 적용

#### 4.2 사이드바 테마 적용

**Steps:**

1. 특정 테마 선택
2. 사이드바 영역 테마 확인

**Expected Results:**

- 사이드바 배경색이 테마에 맞게 변경
- 폴더, 페이지 목록 색상 테마 적용
- 아이콘 색상 테마에 맞게 변경

## 테스트 환경

- 로그인 필요: 테마 저장 시 예 (비로그인 시 로컬 스토리지 사용)
- 다양한 화면 크기에서 테스트 권장
